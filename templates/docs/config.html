Config: Configure the permission system.

Disable: Disable commands for users, roles, and channels.

Usage: -disable [entity] <commands>
Permission: Manage Server
Output: Prevents specific commands from being run in channels, by users, or by any user with a given role.
Notes: If no 'entity' is specified, the command will be disabled for the whole server. Users with Manage Server can still use the bot, regardless of whether or not a command has been disabled. Use -disable all as a shorthand to disable all commands.
Explanation: If channels are passed, any commands passed will not be available to users inside that channel. If roles are passed, any user with that role will not be able to execute the passed commands in all channels in the server. If a user is passed, they will not be able to run any passed commands inside of the server.
Subcommands: -disable list # List all the disabled commands. -disable clear # Unignore all entities.

Disable Clear: Clear the disabled command list.

Usage: -disable clear
Permission: Manage Server
Output: Enables all the disabled commands for channels, roles and users.

Disable List: Show disabled channels, roles, and users.

Usage: -disable list [entity] [dm]
Permission: Manage Server

Enable: Enable commands for users, roles, and channels.

Usage: -enable [entity] <commands...>
Permission: Manage Server
Output: Prevents specific commands from being run in channels, by users, or by any user with a given role.
Notes: If no 'entity' is specified, the command will be enabled across the whole server. Users with Manage Server can still use the bot, regardless of whether or not a command has been disabled. Use -disable all as a shorthand to disable all commands.
Explanation: If channels are passed, any commands passed will not be available to users inside that channel. If roles are passed, any user with that role will not be able to execute the passed commands in all channels in the server. If a user is passed, they will not be able to run any passed commands inside of the server.
Subcommands: -disable list # List all the disabled commands. -disable clear # Unignore all entities.

Enable All: Enable all disabled commands

Usage: -enable all
Permission: Manage Server
Output: Enables all the disabled commands for channels, roles and users.
Notes: Alias for -disable clear

Ignore: Ignore channels, roles, and users.

Usage: -ignore [entities...]
Aliases: -restrict, -plonk
Permission: Manage Server
Output: Prevents commands from being run in channels, by users, or by any user with a given role.
Notes: Users with Manage Server can still use the bot, regardless of whether or not they've been ignored. Use -ignore all as a shorthand to ignore everyone.
Explanation: If channels are passed, no commands will be available to users inside that channel If roles are passed, any user with that role will not be able to execute commands in all channels in the server. If a user is passed, they will not be able to run any commands inside of the server.
Subcommands: -ignore list # List all the ignored entities. -ignore all # Shortcut for -ignore @everyone. -ignore clear # Unignore all entities.

Ignore All: Ignore all server users.

Usage: -ignore all
Permission: Manage Server
Output: Ignores every single user in the server without the Manage Server permission
Notes: Shortcut for -ignore @everyone Useful for not mentioning @everyone

Ignore Clear: Clear the ignore list.

Usage: -ignore clear
Permission: Manage Server
Output: Removes all entities from the ignored list of objects.

Ignore List: Show all ignored objects.

Usage: -ignore list Permission Manage Server
Output: Shows all ignored users, roles, and channels in the server

Unignore: Unignore channels, users, and roles.

Usage: -unignore [entities...]
Alias: -unplonk, -unrestrict
Permission: Manage Server
Output: Allows channels, users, or roles to run bot commands again.
Subcommands: all # Unignore all previous entities.

Unignore All: Unignore all users, roles, and channels

Usage: -unignore all
Permission: Manage Server
Output: Unignores all users, roles, and channels that were ignored.
Notes: An alias for -ignore clear

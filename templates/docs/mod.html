Mod: Moderate server users.

Addrole: Add multiple roles to a user.

Usage: -addrole <user> [roles]...
Aliases: -ar, -addroles
Permission: Manage Roles
Output: Adds multiple roles to a user
Notes: If the role is multiple words, it must be surrounded in quotations. e.g. -ar Hecate "this role"

Ban: Ban users from the server.

Usage: -ban <targets>... [delete message days = 1] [reason = "No reason"]
Example: -ban @Jacob Sarah 4 for advertising
Permission: Ban Members
Output: Ban passed members from the server.

Blind: Hide a channel from a user.

Usage: -blind <target> [target]...
Example: -blind Hecate 708584008065351681 @Elizabeth
Permission: Kick Members
Output: Prevents users from seeing the channel.

Block: Restrict users from sending messages.

Usage: -block <target> [target]...
Example: -block Hecate 708584008065351681 @Elizabeth
Permission: Kick Members
Output: Stops users from messaging in the channel.
Notes: This mutes users in one channel

Kick: Kick users from the server.

Usage: -kick <target> [target]... [reason]
Example: -kick @Jacob Sarah for advertising
Permission: Kick Members
Output: Kicks passed members from the server.

Lock: Prevent messages in a channel.

Usage: -lock [channel] [duration]
Aliases: -lockdown, -lockchannel
Permission: Administrator
Output: Locked channel for a specified duration. Infinite if not specified

Mute: Mute users for a duration.

Usage: -mute <users>... [duration] [reason]
Alias: -tempmute
Permission: Kick Members
Output: Mutes multiple users. This command will attempt to remove all the roles from the passed users, and reaasign them on unmute. If no duration is specified, mute will be indefinite.
Notes: Duration and reason are optional. Running the command with a reason will dm the user while running the command without a reason will not dm. Run the
command: -examples mute for specific usage examples.

Removerole: Remove multiple roles to a user.

Usage: -removerole <user> [roles]...
Aliases: -rr, -rmrole, -remrole
Permission: Manage Roles
Output: Removes multiple roles to a user
Notes: If the role is multiple words, it must be surrounded in quotations. e.g. -rr Hecate "this role"

Slowmode: Set the slowmode for a channel

Usage: -slowmode [channel] [seconds]
Permission: Manage Channels
Output: Sets the channel's slowmode to your input value.
Notes: If no slowmode is passed, will reset the slowmode.

Softban: Softban users from the server.

Usage: -softban <targets> [delete message = 7] [reason]
Example: -softban @Jacob Sarah 6 for advertising
Permission: Kick Members
Output: Softbans members from the server.
Notes: A softban bans the user and immediately unbans them in order to delete messages. The days to delete messages is set to 7 days.

Tempban: Temporarily ban users.

Usage: -tempban <users> [duration] [reason]
Alias: -tban
Output: Temporarily bans a member for the specified duration. The duration can be a a short time form, e.g. 30d or a more human duration like "until thursday at 3PM".

Temprole: Temporarily add roles to users.

Usage: -temprole <user> <duration>
Alias: -trole
Output: Adds a role to a user for the specified duration. The duration can be a a short time form, e.g. 30d or a more human duration like "until thursday at 3PM".

Unban: Unban a previously banned user.

Usage: -unban <user> [reason]
Alias: -revokeban
Example: Unban Hecate#3523 Because...
Permission: Ban Members
Output: Unbans a member from the server.
Notes: Pass either the user's ID or their username

Unblind: Reallow users see a channel.

Usage: -unblind <targets>...
Example: -unblind Hecate 708584008065351681 @Elizabeth
Permission: Kick Members
Output: Reallows blinded users to see the channel.

Unblock: Reallow users to send messages.

Usage: -unblock <target> [target]...
Example: -unblock Hecate 708584008065351681 @Elizabeth
Permission: Kick Members
Output: Reallows blocked users to send messages.
Notes: This unmutes users in one channel

Unlock: Unlock a channel.

Usage: -unlock [channel]
Aliases: -unlockchannel, -unlockdown
Output: Unlocks a previously locked channel.

Unmute: Unmute muted users.

Usage: -unmute <users>... [reason]
Output: Unmutes previously users previously muted by the -mute command. The bot will restore all the users' old roles that they had before they were muted.
